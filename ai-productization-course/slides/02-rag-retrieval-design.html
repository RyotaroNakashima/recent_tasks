02-rag-retrieval-design

Retrieval設計：RAG品質の上限を決める層

実務で説明・設計・評価・運用できる力へ接続する章です。

操作: ← / → キー、または下部ボタンで移動。

学習ゴール

  • retrievalが回答品質の上限を決める理由を説明できる。
  • chunking、embedding、vector DB、metadata、filtering、rerankingを使い分けられる。
  • OCR由来テキストのノイズを考慮した検索設計ができる。

なぜ学ぶのか

本番RAGでは、部品の名前を知るだけでは不十分です。ユーザー要件、データ特性、失敗時の影響から、どの設計を選ぶかを判断します。

retrievalが上限を決める理由

生成モデルは渡されなかった事実を正しく引用できません。欲しい文書がtop-kに入らなければ、どれだけpromptを工夫しても正答率は頭打ちになります。

chunking設計の選択肢

  • 固定長chunk: 実装容易だが意味境界を壊す。
  • 見出しベースchunk: 規程・設計書に強い。
  • semantic chunk: 意味まとまりを優先。
  • parent-child chunk: 小chunkで検索し親文書で回答。
  • sliding window: 文脈欠落を減らすが重複コスト増。

embeddingモデルの選び方

日本語、専門用語、短文/長文、コスト、更新頻度で選びます。OCRノイズが多い場合は、表記揺れ・誤字への耐性を評価セットで比較します。

vector DBの役割

vector DBは近傍検索だけでなく、metadata filter、namespace、index更新、削除、監査を担います。PoCではFAISS、本番では運用機能を含めて選定します。

metadata設計とfiltering

metadata = {source_type, department, created_at, updated_at,
            access_group, customer_id, ocr_confidence, verified}

権限・鮮度・データ種別は検索前filterで絞るのが原則です。

hybrid searchとreranking

BM25は固有名詞や型番に強く、embeddingは意味類似に強い。hybridで候補を広げ、rerankerで質問との関連度を再評価します。

query rewritingとmulti-query retrieval

ユーザー質問を検索向けに言い換えたり、複数観点のqueryを発行します。ただしログと評価なしに増やすとコストとノイズが増えます。

access controlを考慮したretrieval

権限外文書は生成後に隠すのではなく、retrieval前にfilterします。検索結果に入った時点で情報漏えいリスクがあります。

OCR由来テキストのノイズ対策

  • ocr_confidenceが低いchunkはrankingを下げる。
  • 会社名・メールなどは正規化フィールドも保持。
  • 画像座標やページ番号をcitationに残す。
  • 人手確認済みフラグをmetadataに入れる。

retrieval失敗パターン

  • 欲しい文書が取れない: chunk粒度・query・embeddingを見直す。
  • 関係ない文書が混ざる: rerankingとmetadataを改善。
  • 古い文書を取る: updated_atをrankingへ反映。
  • 権限外文書を取る: access_group filterを必須化。
  • 似ているが違う文書を取る: 固有IDや部署でfilter。

OCR/画像AI経験との接続

OCRの前処理、信頼度、誤差分析、項目一致率の考え方は、RAGの文書整形、metadata、failure taxonomy、評価設計へ転用できます。

面接・職務経歴書での語り方

「モデル精度だけでなく、データ品質・検索品質・生成の忠実性・運用監視まで分解して改善できます」と、担当範囲をプロダクト品質に広げて表現します。

ミニ演習

社内FAQ、議事録、設計書、OCR文書を混ぜたRAGについて、source_type、権限、更新日、顧客ID、OCR信頼度を含むmetadata設計を作る。

まとめ

  • 技術要素は必ず業務要件と失敗リスクに紐づけて選ぶ。
  • RAG品質は検索、生成、評価、運用の掛け算で決まる。
  • OCR経験者はノイズと評価を扱える点が強みになる。

確認問題

  1. この章の主要技術を3つ挙げ、それぞれの実務判断を説明してください。
  2. 典型的な失敗例を2つ挙げ、検知方法と改善策を述べてください。
  3. OCR/画像AI経験と接続できる点を1つ説明してください。
