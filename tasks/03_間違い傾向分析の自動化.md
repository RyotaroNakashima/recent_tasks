# 間違い傾向分析の自動化

## 概要
名刺画像、GT、AIOCRの推論結果をVLMに渡し、OCR誤りの種別と要因を出力させるテーマ。本人の主担当。2026-03-31での一区切り想定だったが、VLM入力向けの前提整理が不足しており、追加検証が必要になっている。

## 現状の事実
- このテーマは本人が担当している。
- やりたいことは、名刺画像、GT、AIOCRの推論結果と一緒に、誤り種別と誤り要因の一覧をVLMへ与え、OCRが間違った理由を出力させること。
- 当初は、2026-03-31でクローズする想定があった。
- 修報までにやりたかったのは、誤り種別 / 要因の初期例示を拡充し、その状態のプロンプトで人手評価結果と突き合わせて性能確認すること。
- 実際には、例示拡充の前提となる整理で追加課題が見え、修報時点では人手評価との突き合わせまで間に合わない認識になっている。
- 既存の誤り種別 / 要因一覧は、人手向け表現で作られている認識がある。
- カラム名が短く、説明が少ない。
- そのままだとVLMに意図が伝わりにくそう。
- かといって長文説明を入れると、入力コンテキスト長の制約にかかりそう。
- そのため、短文で意味を損なわない表現へ再設計する必要があるという認識がある。
- 入力ルール起因の誤り要因は細かすぎて、そのまま渡してもVLMが追従できないのではという懸念がある。
- 主要な要因だけ渡すか、発生頻度の高い要因だけ渡すかなど、与え方の追加検証が必要という考えがある。
- `その他` `原因不明` などのまとめ先も作りたい認識がある。
- 2026-04以降に延長したいが、どの課題にどのくらい工数がかかるかはまだ曖昧。

## いまの仮方針
- 今回の修報では、`評価完了` ではなく `評価可能な入力定義の整理フェーズ` として説明する。
- 2026-03-31で無理にクローズせず、追加検証を継続する。
- 延長理由は、少なくとも次の2本で説明する。
- 誤り種別 / 要因一覧をVLM入力向けに再設計する必要が見えた。
- 入力ルール系の要因をどの粒度でどこまで与えるべきか、追加検証が必要と分かった。
- 次の修報までに、課題整理と具体スケジュールを提示する方向がよさそうという考えがある。

## 引っかかり・止まりポイント
- 2026-03-31で終わる予定だったのでは、と突っ込まれる不安が強い。
- 例示拡充は一見すぐ終わりそうに見えたが、前提整理が重く、見積もりが甘かった感覚がある。
- 入力ルール要因をどの程度まで単純化するかの判断がまだ固まっていない。
- ガントチャートや延長期間を今すぐ言い切れないことが、修報での説明不安につながっている。
- `進捗がない` のではなく `やってみて追加課題が見えた` 状態だが、その言い換えを意識的にしないと後ろ向きに見えやすい。
